3. Accessibility: Color Contrast

Text needs sufficient contrast against its background to be readable, especially for users with visual impairments (WCAG guidelines define minimum ratios).

Good Contrast Examples

Black text on white background (High Contrast)
White text on dark blue background (Good Contrast)
Black text on light yellow (Acceptable Contrast)

Poor Contrast Examples

Light gray text on white (Very Poor Contrast)
Dark blue text on medium blue (Poor Contrast)
Light yellow text on medium yellow (Poor Contrast)

**Why?** Poor contrast makes text difficult or impossible to read for many users. Use online contrast checker tools to verify your color choices meet WCAG AA standards (usually a 4.5:1 ratio for normal text).

4. Accessibility: ARIA & Interactivity (Alpine.js)

ARIA attributes help make custom interactive elements understandable to assistive technologies (like screen readers). Alpine.js can manage the state and update ARIA attributes dynamically.

Example: Disclosure Widget (Show/Hide Panel)

Compare the two implementations below using a screen reader or keyboard navigation.

❌ Without Proper ARIA

Toggle Details (Non-semantic)
This content panel is shown or hidden, but its state isn't clearly communicated to screen readers, and the 'button' isn't easily focusable or identified as a button.

This uses a <div> as a button (bad practice) and lacks aria-expanded/aria-controls.

✅ With Proper ARIA & Alpine.js

class="border p-3 bg-gray-50 text-sm" > This content panel uses x-show. The button correctly uses aria-expanded (state bound to Alpine's expanded variable) and aria-controls (linking button to panel). Screen readers announce the state (expanded/collapsed).

Uses a real <button>, aria-expanded, and aria-controls linked via id.

**Why ARIA?**

  • <button>: Natively focusable and announced as a button.
  • aria-expanded="true/false": Tells screen readers if the controlled content is currently visible or hidden. Alpine dynamically sets this (:aria-expanded="expanded.toString()").
  • aria-controls="ID_REF": Links the button to the specific panel it controls (using the panel's id).
  • id="details-panel-aria": Provides the target for aria-controls.

Alpine.js (x-data, @click, x-show, attribute binding :aria-expanded) makes managing the state and ARIA attributes straightforward.

5. DOM (Conceptual)

The DOM is the browser's internal representation of the HTML page structure. JavaScript (including libraries like Alpine.js and HTMX) interacts with the DOM to dynamically change what you see on the page.

<div id="container">
  <p class="text-blue-500">Hello</p>
</div>

Think of the HTML above as a blueprint. The browser builds an object structure (the DOM) from it. JavaScript can then find elements (e.g., the <div> with `id="container"`) and modify them (e.g., change the text inside the <p>, add/remove classes like `text-blue-500`, or hide/show the whole `div`). Alpine.js and HTMX provide convenient ways to declare these DOM manipulations directly in your HTML.
